Available for macOS, Linux, and Windows Web developers often log messages to the Console to make sure that their JavaScript is working as expected. To log a message, you insert an expression like console.log('Hello, Console!') into your JavaScript. When the browser executes your JavaScript and sees an expression like that, it knows that it's supposed to log the message to the Console. Web developers log messages for 2 general reasons: Making sure that code is executing in the right order. Inspecting the values of variables at a certain moment in time. The Console is also a REPL. You can run JavaScript in the Console to interact with the page that you're inspecting.
